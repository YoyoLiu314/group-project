Data Science Project Plan¶

(1) Data Description¶

The dataset utilized in this project is players.csv. Data inconsistencies include missing values, incorrect formats, and recruitment biases affecting representativeness. Self-reported data may be biased, influencing player behavior and data collection, requiring verification for reliable analysis.

Players Data¶

The players dataset contains specific information about each Minecraft user. This dataset includes 196 observations representing each player and 7 variables describing their characteristics.

  • experience: The data type is chr. This variable categorizes each player into a 'title' representing their level of experience.
  • subscribe: The data type is lgl. This variable tells us whether the player is subscribed to a game-related newsletter using TRUE or FALSE.
  • hashedEmail: The data type is chr. This lists out the hashed (anonymized) emails of the players.
  • played_hours: The data type is dbl. This tells us how many hours each player played Minecraft.
  • name: The data type is chr. This tells us the players' name.
  • gender: The data type is chr. This categorizes the players into genders they identify with.
  • Age: The data type is dbl. This tells us the players' age.

(2) Questions¶

Broad Question:¶

What player characteristics and behaviours are most predictive of subscribing to a game-related newsletter, and how do these features differ between various player types?

Specific Question:¶

Can a player's played hours predict if they subscribe to a game-related newsletter?

How the data addresses the question:¶

  • Dataset used: players_csv
  • The explanatory variable: played_hours in the players.csv dataset.
  • played_hours: This variable will help determine whether spending longer hours playing Minecraft influence subscription.

Brief description of data wrangling¶

  1. We would need to clean the data to check for missing values in the relevant columns (played_hours, subscribe) and ensure the data types are correct and consistent.
  2. The relevant columns need to be selected (played_hours, subscribe) and plotted to view the relationship between the two.
  3. A classification model (KNN) is chosen to predict the outcome of whether players are likely to subscribe to game-relatd newsletters based on the hours of Minecraft they play.

(4) Methods and Plan¶

The chosen model is k-Nearest Neighbours (KNN) for classification. Since the data’s relationship is unknown, KNN is ideal as it makes no assumptions about form. It compares subscribe (TRUE/FALSE) based on played hours. Data scaling is needed to prevent large values from dominating distance calculations. Selecting an optimal k value requires cross-validation to avoid overfitting or underfitting. Performance metrics (accuracy, precision, recall) will evaluate the model. The dataset will be split 75%-25% for training and testing, with cross-validation applied to the training set to determine the best k value.
